Centrifuge App

A decentralized web app for investors to invest crypto, and for issuers to finance real-world assets

k/f dev AG, Jan – Aug 2021
Design of a new app: Wireframing, visual design, prototyping, concept validation, UI specification, quality assurance

How might we recreate the Tinlake ↗ app on Ethereum for the Centrifuge blockchain and include functionality to allow issuers to originate, finance and repay their assets?

We started the design process by creating the scope outline, defining the first-level objects, and grouping the functionality by user groups, i.e. investors, issuers, and underwriters. Since the code audit required a user interface, we created wireframes to implement a preliminary, not feature-complete proof-of-concept (POC) version to test crucial functionality, like investing in a token, creating a pool of assets, or minting an NFT to originate an asset.

A wireframe as preliminary proof-of-concept

To create a design concept covering the entire scope, we created clickable Figma prototypes of the main tasks of each user group, investors and issuers:

  • Investors: find investment, and invest & redeem token
  • Issuers: Create a pool, create an asset, finance & repay the asset

Testing the prototypes with users allowed us to validate the design concept against the information architecture, content, and visual design.

The tested design of the pool detail

We analyzed and prioritized the findings to then solve the identified pain points.

The improved design of pool detail

The result is a single-page app using a three column-layout hosting a global navigation sidebar on the left, a main column for the selected content in the middle, and a right-hand toolbar for wallet-address specific transactions.
